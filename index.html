關於我

我是一位致力於以「人」為核心的設計實踐者，關注設計如何在真實生活中創造價值與情感連結。 我的設計興趣橫跨產品設計、服務設計與使用者經驗（UX），並深信設計不僅是形式的創造， 更是一種系統性解決問題的方式。

我擅長運用多種研究方法，從觀察與訪談中精準提煉使用者需求，並將複雜的議題結構化。 在設計過程中，我特別重視跨領域整合，並透過模型製作、互動原型與視覺呈現， 將抽象概念具體落實。

我期許在學術研究中，持續探索人與環境的互動模式，創造出具備深度、機能與未來感的設計提案。

Performance Indicators

持續完成設計課程專題、模型製作與設計實作

系統性技術學習

透過線上課程與教學影片，建立完整的技術知識體系

  • 前端基礎：深入學習 HTML5 語義化標籤、CSS3 進階特性（Flexbox、Grid、動畫）、JavaScript ES6+ 語法與 DOM 操作
  • 後端基礎：理解資料流程、API 設計概念、前後端串接機制、資料庫基本操作與 RESTful API 設計原則
  • 實作練習：完成多個課程專案與實作範例，從基礎到進階逐步累積實際開發經驗，建立程式碼除錯與問題解決能力

提前適應研究所學習

大四期間主動先修研究所課程，提前熟悉學術研究的學習節奏與課程深度

  • 參與研究所層級的設計理論課程，學習系統性研究方法與學術寫作規範
  • 提前接觸互動設計、使用者研究等進階課程，建立更深入的設計思維
  • 適應高強度的學術討論與專案要求，培養獨立研究與批判性思考能力

持續設計實作

在學習技術的同時，持續完成設計課程專題、模型製作與設計實作

  • 完成多個設計專題，從概念發想到原型製作，完整實踐設計流程
  • 精進模型製作技能，熟悉各種材料特性與加工技術，提升實體原型品質
  • 整合設計與技術，將設計概念透過程式碼實現，驗證設計的可行性與使用者體驗

Strategic Objectives

不只停留在設計表現，而是補足數位與技術理解

補足數位與技術理解

突破傳統設計師的框架，主動學習數位技術與程式開發知識

  • 理解前端開發流程，掌握 HTML、CSS、JavaScript 等基礎技術，能夠獨立完成簡單的網頁實作
  • 學習後端邏輯與資料處理概念，理解系統架構與 API 運作機制，提升對數位產品的整體認知
  • 熟悉設計工具與開發工具的整合，了解設計稿到實際產品的轉換過程與技術限制

建立研究所學習基礎

為研究所階段深入學習互動設計、使用者研究與系統型設計課程建立扎實基礎

  • 互動設計：學習介面互動模式、動態效果設計、使用者流程規劃，理解數位產品的互動邏輯與體驗設計
  • 使用者研究：掌握質性與量性研究方法，學習訪談技巧、問卷設計、使用者測試方法，建立以使用者為中心的設計思維
  • 系統型設計：學習系統思考方法，理解複雜問題的結構化分析，培養從整體角度思考設計解決方案的能力

提升設計落地可行性

讓設計不只停在概念階段，而能與工程、技術團隊有效溝通，理解實際實作限制，提升提案落地的可行性

  • 技術溝通能力：學習技術術語與開發流程，能夠與工程師進行有效溝通，理解技術可行性與實作複雜度
  • 實作限制理解：了解前端框架、後端架構、資料庫設計等技術限制，在設計階段就考慮實作可行性
  • 提案優化：基於技術理解優化設計提案，提出既符合設計目標又具備實作可行性的解決方案，提升設計落地的成功率

IDP

以「人」為核心，真正理解使用者需求，而非只做形式

深度使用者研究能力

掌握多種研究方法，從觀察與訪談中精準提煉使用者需求，建立以使用者為中心的設計思維

  • 質性研究：熟練運用深度訪談、焦點團體、使用者觀察等方法，深入理解使用者的真實需求、痛點與行為模式
  • 量性研究：學習問卷設計、數據分析、統計方法，透過數據驗證設計假設，建立客觀的設計決策依據
  • 需求分析：將複雜的使用者反饋結構化，識別核心需求與次要需求，建立清晰的使用者需求層級與優先順序

系統觀點與概念轉化

建立系統性思考能力，將抽象概念轉化為可實現的產品或服務

  • 系統思考：學習系統分析方法，理解問題的整體結構與各要素間的關係，從系統層面思考設計解決方案
  • 概念具體化：將抽象的使用者需求與設計概念，透過使用者旅程、服務藍圖、資訊架構等工具，轉化為具體的設計方案
  • 原型驗證：運用低保真與高保真原型、互動原型、實體模型等工具，驗證設計概念的可行性與使用者接受度

跨領域溝通與設計落地

成為一位能跨領域溝通、理解技術限制、讓設計真正落地的設計師

  • 跨領域溝通：學習與工程師、產品經理、行銷團隊等不同專業背景的成員有效溝通，用對方理解的語言表達設計理念
  • 技術理解：理解前端、後端、資料庫等技術架構，了解技術限制與可能性，在設計階段就考慮實作可行性
  • 設計落地：從概念發想、原型製作到實際開發，全程參與設計實現過程，確保設計理念能夠完整落地，創造真正的使用者價值
